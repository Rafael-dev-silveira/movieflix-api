.Crie uma pasta para o projeto chamada movieflix-api,aperte ctrl + ''(aspas) e abra o terminal
do vscode(git bash),e coloque :


git init                //para inicializarmos o projeto como um repositorio git.

npm init -y           //cria o package.json,o -y é para criar padrão,sem fazer preenchimentos pelo usuario

.Agora crie uma pasta src ,e nela um arquivo server.ts (vamos fazer com typeScript) .

.Para usar o typeScript com node vamos fazer algumas configurações,o node não foi feito usando
o typeScript ,ele não vai entender o código TS por padrão,agente vai ter que converter este código
para código javaScript,assim os servidores que estão rodando o node vão passar a entender
o nosso código se estiverem com javaScript.


.Vamos concerter código typeScript para javaScript .

.Primeiro passo é criar um arquivo de configuração do typeScript aqui no nosso projeto.


.No terminal do vscode :


npm i -D typescript


*-D = D maiusculo ,instala o typeScript como uma dependencia de desenvolvimento,ou seja,só
vamos usar enguanto estivermos desenvolvendo ,não vai para a produção.


----------------------------------------------------------------


.Agora vamos criar a configuração typeScript :

.No terminal coloque :


npx tsc --init              //executar comandos do typeScript,e iniciar nosso arquivo de configuração


*npx = este x no final significa executar,então o npx vai executar alguma ferramenta de linha
de comando dentro da nossa aplicação.


*Agora o arquivo foi criado tsconfig.json ,agora na linha de comando vamos usar o npx para
executar.


*Então o npm é o gerenciador de pacotes,o npx é um executor de pacotes.



*Tive aquele problema novamente,no Meu Drive(google drive),não baixou tudo e deu erros,
devido ao nome no inicio estar com espaço etc,coloquei o projeto no onedrive,nesta pasta
dev-em-dobro ,e deu tudo certo.


----------------------------------------------------------------------------------------------



.Agora abra o arquivo json que criamos, tsconfig.json .

.Nele vai ter a maioria das coisas comentadas,quer dizer que não é muito importante,
mas as que estão descomentadas são importantes para nós,vamos configuralas,o target é 
importante vamos trocar a versão dele :


"target": "es2019",



*Vai ficar assim,olhe no link da documentação,lá aparece a versão mais nova.
*Documentação target do tsconfig: https://www.typescriptlang.org/pt/tsconfig#target

.No server.ts :




const http = require('http');      //vai aparecer um erro,até instalar o @types/node

const server = http.createServer((req, res) => {
   res.setHeader('Content-Type', 'text/plain');
 
   if (req.url === '/') {
       res.statusCode = 200;
       res.end('Home page');
   } else if (req.url === '/sobre') {
       res.statusCode = 200;
       res.end('About page');
   }
});

server.listen(3000, () => {
 console.log(`Servidor em execução em http://localhost:3000/`);
});



*Esse código não vai funcionar,por que ainda não instalemos um pacote ,para node entender TypeScript.


.No terminal :


npm i -D @types/node             //instala como dependencia de desenvolvimento,olhe no package.json



npx tsc src/server.ts


*Esse comando vai usar o typeScript usando o npx para executar o typeScript para ele
converter em código javaScript.

*Agora ele converteu e criou um arquivo JS server.js,com o nosso código todo compilado.


.Para rodar o nosso servidor :


node src/server.js


*Agora é só segurar ctrl + click na palavra localhost,nosso servidor está funcionando.


*Mas para fazer essa configuração de typeScript para javaScript é muito demorada e custosa,
vamos usar uma maneira de facilitar as coisas usando o TSX .


*O TSX vai converter automaticamente de typeScript para javaScript ,isso tudo sem precisarmos
fazer tantos comandos como antes.



-----------------------------------------------------------------------------------------------



.Projeto com Node e PostgreSQL - Configurações iniciais - Parte 2


.Nessa aula vamos continuar as configurações do nosso projeto, começando com a instalação e utilização 
do TSX. Que é responsável pela conversão do código TS em JS.


.Agora vamos instalar o TSX ,no terminal coloque : ctrl + c ,para parar o servidor no terminal :


npm i -D tsx                 //olhe ele no package.json




*Agora voce pode excluir o arquivo server.js e usar o server.ts .


----------------------------------------------------


.Agora como rodar o TSX sempre que alterarmos o código :



npx tsx watch src/server.ts



*npx tsx = executar o comando tsx

*watch = vai ficar observando alterações no arquivo,quando alterarmos algo no código ele vai
fazer automaticamente a conversão de typeScript para javaScript.

*src/server.ts = o caminho do arquivo que queremos compilar.


*Agora aperte ctrl + click em localhost,e abre nosso servidor,qualquer alteração que voce fizer
no código,vai ser atualizado automaticamente pelo tsx,sem precisar colocar os comandos toda vez,
no mais só atualize a página .



-----------------------------------------------------------



.Agora abra o arquivo package.json e coloque :



  "scripts": {
    "dev":"npx tsx watch src/server.ts"        
  },


*Criei essa variavel "dev" e o comando está todo dentro dela,para min não precisar colocar
esse nome enorme toda vez que eu for rodar,agora posso colocar só :


npm run dev          //está aqui o comando todo em dev .



---------------------------------------------------------------------



.Agora vamos rodar um comando do git para nós podermos subir estas alterações para o nosso github.

.No arquivo server.ts pode apagar todo o código,era só para testar e ver o TS sendo compilado para JS.

.Crie um arquivo chamado .gitignore ,para ignorar alguns arquivos que não queremos que subão junto
com nossos arquivos do projeto para o github.

.Nós nunca mandamos a pasta node_modules para o github,então abra o arquivo .gitignore e coloque :


/node_modules       //agora esta pasta não vai subir junto com nossos arquivos para o github



*Note que antes disso no icone do vscode source control(fonte de controle) estava um numero de
295 alterações,e depois de ignorar a pasta node_modules ele passa a ter só 5 alterações,
por que está pegando só os arquivos do nosso projeto.





------------------------------------------------------------------


.Agora vamos rodar o push,e adicionar estas alterações ,coloque :


git add .

git commit -m "configurações iniciais do projeto"    //essa frase coloquei o que estou fazendo
